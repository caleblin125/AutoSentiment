  What you're building: The FastAPI server that takes HTTP requests, manages runs in the database, streams events to the browser, and
   (last) wires all the other tracks together into the end-to-end pipeline.

  Files you own: app/api/routes.py, app/agents/orchestrator.py, app/reports/builder.py
  Do not touch: anything in agents/ollama.py, agents/nemoclaw.py, agents/light_queue.py, tools/, ingest/

  Order of work: event bus → routes → report builder → orchestrator (orchestrator is last because it calls B and C's code)

  Prompt:
  Read SPEC.md sections: API, SSE Event Types, SSE Internal Event Bus, Report Structure, Team Tracks §Track A.

  The file app/api/event_bus.py is already implemented — use it as-is.
  The SQLAlchemy models (Run, EvidenceChunk, RunEvent) are in app/models/__init__.py.
  The DB session factory and get_db dependency are in app/db/session.py.

  Implement the following, in this order:

  1. app/api/routes.py — all 4 routes:
     - POST /api/runs: validate body (topic, optional freshness), create Run row with status="pending",
       register an event queue via event_bus.register(run_id), launch asyncio.create_task(run_research(...)),
       return {"run_id": run.id}
     - GET /api/runs/{run_id}: return run row as dict including report field
     - GET /api/runs/{run_id}/events: SSE StreamingResponse — get queue via event_bus.get(run_id),
       yield "data: {json}\n\n" for each item from the queue, stop on None sentinel,
       call event_bus.deregister on exit
     - GET /api/runs/{run_id}/evidence/{chunk_id}: return EvidenceChunk row as dict

  2. app/reports/builder.py — pure Python, no LLM:
     - compute_counts(chunks): tally label counts by overall and by source_type, return ratios as floats
     - pick_top_quotes(chunks, label, n=5): return up to n {summary, evidence_id, url} dicts
     - build_report stub (will call synthesize_report from nemoclaw.py — leave as NotImplementedError for now)

  Do not implement orchestrator.py yet — that comes after the data and LLM tracks are done.
  Use async SQLAlchemy throughout. Refer to app/db/session.py for the session pattern.

  Track B — Data Pipeline

  What you're building: Everything that touches the web — searching Brave, fetching Reddit JSON, extracting news articles. You
  produce FetchedItem objects that the orchestrator feeds to the LLM tier.

  Files you own: app/tools/search.py, app/ingest/fetch.py
  Do not touch: anything in agents/, api/, reports/

  Prompt:
  Read SPEC.md sections: Sources, Caps & Budgets, Reddit JSON Fields, Team Tracks §Track B.

  The FetchedItem dataclass and is_reddit_url helper are already stubbed in app/ingest/fetch.py.
  The SourceType enum is in app/agents/types.py.
  Settings (brave_api_key, max_urls_per_run, ollama_base_url) come from app/core/config.py.

  Implement the following:

  1. app/tools/search.py — brave_search(query, *, freshness, count, settings) -> list[str]:
     - POST to https://api.search.brave.com/res/v1/web/search
     - Headers: {"Accept": "application/json", "X-Subscription-Token": settings.brave_api_key}
     - Params: {"q": query, "count": count} + {"freshness": freshness} if freshness is set
     - Rate limit: acquire _rate_sem, make the call, then await asyncio.sleep(1), then release
     - Return list of URLs from results[].url

  2. app/ingest/fetch.py — three functions:
     - _fetch_reddit(url): httpx GET url.json (add .json if missing), parse
       response[1]["data"]["children"], filter kind=="t1", extract child["data"]["body"],
       skip empty bodies, cap at REDDIT_COMMENTS_PER_THREAD, return list[FetchedItem]
     - _fetch_news(url): httpx GET with a browser User-Agent header, pass html to
       trafilatura.extract(), split on "\n\n", filter chunks under 40 chars, return list[FetchedItem]
     - fetch_items(url): call is_reddit_url(url) → dispatch to reddit or news fetcher.
       Catch all exceptions and return [] so one bad URL doesn't kill the run.

  Use httpx.AsyncClient with a 10-second timeout. Test each function with a real URL using
  asyncio.run() before calling it done.

  What you're building: The two Ollama model integrations — the 30B model that labels every single piece of content, and the 120B
  model that expands search queries and writes the final narrative. Everything that touches AI lives here.

  Files you own: app/agents/ollama.py, app/agents/light_queue.py, app/agents/nemoclaw.py
  Do not touch: anything in api/, tools/, ingest/, reports/

  Prompt:
  Read SPEC.md sections: Hardware & Inference, LLM Prompt Contracts, Team Tracks §Track C.

  The SentimentResult dataclass and SentimentLabel enum are in app/agents/types.py.
  Settings (ollama_base_url, lightweight_model, nemoclaw_model) come from app/core/config.py.

  Implement the following, in this order:

  1. app/agents/ollama.py — ollama_generate(prompt, *, system, model, base_url) -> dict:
     - POST to {base_url}/api/generate
     - Body: {"model": model, "prompt": prompt, "system": system, "format": "json", "stream": false}
     - Return json.loads(response.json()["response"])
     - Raise ValueError with the raw response text if json.loads fails
     - Use httpx.AsyncClient with a 60-second timeout

  2. app/agents/light_queue.py — SentimentQueue._call_model(snippet) -> SentimentResult:
     - Use the system prompt and prompt template from SPEC.md §30B per-item sentiment
     - Call ollama_generate with settings.lightweight_model
     - Parse label and summary from the returned dict
     - On any exception, return SentimentResult(label=SentimentLabel.NEUTRAL, summary="parse error")

  3. app/agents/nemoclaw.py — two functions:
     - expand_queries(topic, *, settings) -> list[str]:
       Use the prompt from SPEC.md §120B query expansion. Parse the "queries" key.
       On failure, return [topic, topic + " reddit", topic + " review", topic + " news", topic + " opinions"]
     - synthesize_report(topic, chunks_summary, counts, *, settings) -> dict:
       Format chunks_summary as a bulleted list in the prompt (label: summary (source_type)).
       Use the prompt from SPEC.md §120B synthesis. Parse and return {"themes": [...], "narrative": "..."}.
       On failure return {"themes": [], "narrative": "Synthesis unavailable."}

  Test each function standalone with asyncio.run() and a real Ollama instance before handoff.
  Verify both models are loaded: run `ollama list` and confirm both model names appear.

  The Integration Moment

  When B and C are both done and tested, hand off to Track A to wire orchestrator.py. The prompt for that final step:

  Tracks B and C are complete. Implement app/agents/orchestrator.py — run_research(run_id, topic, freshness, settings).

  Follow SPEC.md §Agent Flow exactly:
  1. Set run status="running" in DB. Emit run_started event via event_bus.get(run_id).put_nowait(...)
  2. Call expand_queries(topic, settings=settings) from nemoclaw.py → list of queries
  3. For each query: call brave_search(query, freshness=freshness, settings=settings) from tools/search.py.
     Deduplicate URLs. Stop collecting after settings.max_urls_per_run unique URLs.
  4. For each URL: call fetch_items(url) from ingest/fetch.py. Emit url_fetched. Stop when total
     items across all URLs hits settings.max_items_per_run.
  5. For each item: call await sentiment_queue.analyze(item.snippet). Save EvidenceChunk to DB.
     Emit item_analyzed with evidence_id, label, summary, url, source_type.
  6. Call compute_counts and pick_top_quotes from reports/builder.py.
     Call synthesize_report from nemoclaw.py.
     Assemble full report dict per SPEC.md §Report Structure.
  7. Save report to Run.report, set status="completed". Emit run_completed with the report.
     Put None into the event queue as the SSE sentinel.

  Wrap the whole function in try/except — on any unhandled exception set status="error",
  emit run_error with the message, and put None into the queue.
  Use a single AsyncSession for all DB writes in this function.
